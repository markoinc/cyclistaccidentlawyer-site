## üìå Title Tag Formulas

*Based on Jacky Chou's research and testing*

### Best Practices

1. **Include primary keyword at the beginning** of title
2. **Keep under 60 characters** to avoid truncation
3. **Include location for local SEO** - e.g., "Best Roofer in Vancouver"
4. **Use power words** - "Best", "Top", "#1", "2026"
5. **Match search intent** - transactional vs informational

### Formulas

```
[Primary Keyword] in [Location] | [Brand]
Best [Service] in [City] - [Company Name]
[Number] Best [Product/Service] in [Location] ([Year])
[Primary Keyword] - [Secondary Keyword] | [Brand]
```

### Examples

- `Best Vancouver Roofing | ABC Roofing`
- `10 Best Deck Builders in St. Louis (2026)`
- `Emergency Plumber Near Me - 24/7 Service | Local Plumbing`

## üìë Content Structure & Formatting

*How to structure articles for maximum SEO impact*

### Header Hierarchy

```
H1: Main Keyword (Only ONE per page)
  H2: Major Section Topics
    H3: Subsections
      H4-H6: Further breakdowns
```

### Content Structure Rules

1. **Keyword in first 100 words** - Get your main keyword in the first paragraph
2. **Use subheadings every 200-300 words** - Break up content for readability
3. **Bullet points and numbered lists** - Makes content scannable
4. **Short paragraphs** - 2-3 sentences max for better mobile readability
5. **FAQ sections** - Add FAQ schema for rich results
6. **Table of contents** - For long-form content (1500+ words)

### For Local SEO

- Include NAP (Name, Address, Phone) on every page
- Embed Google Map on contact page
- Use LocalBusiness schema markup
- Include location-specific content

## üîó Internal Linking Strategies

*How to build topical authority through internal links*

### Core Principles

1. **Pillar-Cluster Model** - Main pillar page links to cluster content, cluster links back
2. **Contextual links within content** - Natural links within paragraphs
3. **Every page should have internal links** - Both incoming and outgoing
4. **Use descriptive anchor text** - Include keywords naturally

### Implementation

```
Pillar Page (eg. "Local SEO Guide")
    ‚îú‚îÄ‚îÄ Cluster: "GBP Optimization" ‚Üí links back to pillar
    ‚îú‚îÄ‚îÄ Cluster: "Citation Building" ‚Üí links back to pillar
    ‚îú‚îÄ‚îÄ Cluster: "Review Management" ‚Üí links back to pillar
    ‚îî‚îÄ‚îÄ All clusters link to each other
```

### Best Practices

- Link new content to existing related pages
- Update old content with links to new posts
- Use 3-5 internal links per 1000 words
- Footer and sidebar links for important pages
- Don't overuse exact-match anchor text

## üéØ Keyword Placement Rules

*Where and how to place keywords for maximum impact*

### Primary Keyword Placement

1. ‚úÖ **URL slug** - `example.com/best-roofer-vancouver`
2. ‚úÖ **Title tag** - At the beginning
3. ‚úÖ **H1 heading** - Exactly once
4. ‚úÖ **First paragraph** - Within first 100 words
5. ‚úÖ **Meta description** - Naturally included
6. ‚úÖ **Alt text** - On at least one image
7. ‚úÖ **Subheadings** - In 1-2 H2s

### Secondary Keywords

- Distribute throughout body content
- Use in H3-H4 subheadings
- Include in alt text for other images
- Use in FAQ section questions

### Avoid

- ‚ùå Keyword stuffing
- ‚ùå Exact match keywords in every heading

- ‚ùå Forced, unnatural placement

## ‚ö° Page Speed & Core Web Vitals

*Performance optimization for rankings*

### Core Web Vitals Targets

| Metric | Good | Needs Improvement | Poor |
|--------|------|-------------------|------|
| LCP (Largest Contentful Paint) | <2.5s | 2.5-4s | >4s |
| FID (First Input Delay) | <100ms | 100-300ms | >300ms |
| CLS (Cumulative Layout Shift) | <0.1 | 0.1-0.25 | >0.25 |

### Quick Wins

1. **Image optimization** - Compress, use WebP, lazy load
2. **Minimize CSS/JS** - Remove unused code
3. **Use CDN** - Distribute content globally
4. **Enable caching** - Browser and server-side
5. **Reduce server response time** - Good hosting matters

### Tools

- Google PageSpeed Insights
- Lighthouse (built into Chrome DevTools)
- GTmetrix
- WebPageTest

*From EP 587 - The Complete Rank and Rent Course*

### What is Rank and Rent?
Build and rank a website for specific local keywords, then rent out the site (and its leads) to a business for a monthly fee.

### Pricing Formula

```
Search Volume √ó CTR √ó CR √ó AOV √ó Margins √ó 50%

Example: "vancouver roofing" ‚Üí 2.9k searches/month
2900 √ó 0.3 (CTR) √ó 0.15 (form fill) √ó 0.05 (close rate) √ó $20,000 √ó 0.25 √ó 0.5 = $16k/mo potential
```

### Domain & Website Setup

- **Domains:** Use exact match domains (EMDs)
- **Website:** GeneratePress or simple builders
- **Structure:** Single page sites work for local keywords (very easy!)
- **On-Site SEO:** Use ChatGPT, include keyword in H1s, meta title, FAQ schema + local schema

### Ranking Strategy

**90% of people will rank #1 with:**
1. Citations + foundational links
2. After that: niche edits + BrowserBlast

## üí° Easy Local SEO Wins

*From EP 499 - Quick wins for immediate results*

### 1. Real businesses = easy wins
- Actual brick-and-mortar locations are easiest to rank
- Strong brand signals: Directions CTR, Brand searches
- If you're using fake GMBs... you're probably going to get clapped

### 2. Tons of real reviews = walk in the park
- If a client has way more legit reviews than top snack pack listings, it means the community loves them
- They just need foundational tweaks

### 3. High foot traffic + no rankings = goldmine
- Businesses with real traffic but no visibility
- They're likely one small tweak away from blowing up
- Technical issue ‚Üí fix it ‚Üí rankings in 7 days

### 4. Review optimization = easy lever
- Check if reviews mention target keywords (if not, missed opportunity)
- Are they replying to reviews? If not, fix it
- Prompt customers to drop keywords in their reviews
